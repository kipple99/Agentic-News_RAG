## 시나리오 요약

- 사용자는 국내 전기차/등등 관련 경제 기사만 골라서 수십~수백개를 개인 db로 스크랩해두고 있다.
- 이 db는 사용자가 중요하다고 판단한 고퀄리티 기사만 모여 있지만 최신 이슈나 새로운 기업까지 모두 커버하지는 못한다
- 사용자는 투자 의사결정을 위해 내가 이미 읽어본, 신뢰하는 기사를 우선 참고하고 싶어하지만 필요 시 전체 웹에서 보강정보도 함께 얻고 싶다.
- 이를 위해 먼저 로컬 스크랮 db에서 검색하고, 결과가 부족하면 네이버 search api로 경제뉴스를 추가 검색하는 검색 에이전트를 구현한다

## 고려사항
- 왜 스크랩 DB가 있고(Repeatability & evaluation)
- 왜 거기서 먼저 찾고
- 왜 네이버 fallback 하는지?(1차시도 : 내부 스크랩 DB / 1차시도 실패 시 2차시도 : 네이버 뉴스)
